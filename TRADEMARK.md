# PDK trademark & attribution policy

"PDK" and "Portaldot Dev Kit" are informal project marks used by this
open-source project. There is no registered trademark today.

## What you can do freely

- Refer to this project by name in blog posts, tutorials, videos,
  Discord posts, tweets, and any other unaffiliated discussion.
- Fork the repo, modify the code, and redistribute under the MIT
  license — that's what the license is for.
- Build derivative tools that call PDK as a library or CLI.
- Include the PDK name in a "powered by" or "compatible with" note in
  your own tool.

## What we ask you to avoid

- Naming your own separate tool "PDK" or "Portaldot Dev Kit" in a way
  that would confuse users about which project maintains it.
- Publishing a package on PyPI or npm named `portaldot-pdk` or
  `portaldot-pdk-ts` (or close variants) other than official releases.
- Claiming affiliation with the Portaldot team or the maintainer
  without a written arrangement.

## Fork naming

If you fork and diverge substantially, please rename — e.g. `mypdk`,
`portaldot-tools`, whatever fits — so users can tell the projects
apart. Attribution in your README to the upstream is appreciated but
not required by MIT.

## Contact

Trademark questions or takedown requests: email the maintainer via
the address in [`SECURITY.md`](SECURITY.md), subject prefixed
`[TRADEMARK]`.

This document is informational, not legal advice. If your use case is
close to any of the "avoid" items above, ask first.
