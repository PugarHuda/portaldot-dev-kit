# Code of Conduct

## The short version

Be respectful. Assume good intent. Debate ideas, not people. If in
doubt, ask.

## The longer version

The `pdk` / `pdk-ts` community is small and technical. We optimise for
clear, direct communication about code, chain behaviour, and the
knowledge base. That means:

**Do:**

- Focus critique on the code, the runtime output, the fix step — not
  the contributor
- Ask "why" when a design choice surprises you
- Include reproduction steps in bug reports, even quick ones
- Credit prior work when it inspires yours
- Take ecosystem feedback (upstream `substrate-interface`, `@polkadot/api`,
  PAPI) with the same seriousness you'd want back

**Don't:**

- Personal attacks, harassment, or sustained disruption
- Publish someone else's private key material even as a joke
- Post KB entries that intentionally mislead
- Use the community as a growth channel for unrelated products

## Reporting

If someone's behaviour is unacceptable, email the maintainer:

**hudapugar (at) gmail (dot) com** — subject prefixed `[CoC]`.

Reports are handled confidentially. Enforcement is at the maintainer's
discretion and ranges from a private word to a permanent ban from all
project channels (GitHub, npm, PyPI, Discord, X).
